# EasyTier 配置选项说明

本文说明 `home-mesh` 多设备通用客户端配置中常见开关的含义和建议。

## 推荐通用配置原则

普通手机、笔记本、台式机作为客户端加入网络时，推荐：

- 使用 `dhcp = true`，避免多个设备导入同一配置后虚拟 IP 冲突。
- 使用 `no_listener = true`，避免多个设备占用默认 `11010` 端口。
- 开启 `latency_first = true`。
- 开启 `private_mode = true`。
- 禁用 IPv6：`disable_ipv6 = true`，因为家庭公网节点当前没有公网 IPv6。
- 不启用出口节点、子网代理、系统转发、no-TUN、禁用加密等高级项。

## 家庭接入节点

家庭公网软路由是固定接入节点：

```text
udp://relay.example.net:11010
tcp://relay.example.net:11010
```

家庭节点：

```text
hostname: home-kwrt
virtual ip: 10.144.1.1/24
proxy cidr: 192.168.1.0/24
```

## 各选项说明

### 延迟优先模式：`latency_first`

建议：开启。

作用：路径选择时优先考虑低延迟，而不是只看跳数。适合你的低延迟异地组网目标。

### 用户态协议栈：`use_smoltcp`

建议：普通客户端关闭。

作用：用用户态协议栈处理部分流量，减少对系统防火墙/内核转发配置的依赖。性能和协议兼容性通常不如内核网络栈。更适合无法配置防火墙或特殊受限环境。

### 禁用 IPv6：`disable_ipv6`

建议：当前开启。

原因：家庭公网节点没有公网 IPv6，禁用 IPv6 可减少无效探测和路径误判。手机流量有公网 IPv6时，如果以后要利用 IPv6 P2P，可再关闭该项。

### 自动获取公网 IPv6：`ipv6_public_addr_auto`

建议：关闭。

作用：从共享公网 IPv6 前缀的 peer 自动获取公网 IPv6。当前家庭节点没有公网 IPv6 前缀可共享，不适合开启。

### KCP 代理：`enable_kcp_proxy`

建议：默认关闭。

作用：把虚拟网内 TCP 流量转换为 KCP 传输，可能改善 UDP 丢包环境下的 TCP 体验。它不等于 P2P，也不减少中转带宽。需要双方兼容，按需开启。

### 禁用 KCP 输入：`disable_kcp_input`

建议：默认关闭。

作用：不接受别人用 KCP 代理访问本节点。如果同时开启 KCP 代理又禁用输入，会影响其它节点使用 KCP 访问本机。

### QUIC 代理：`enable_quic_proxy`

建议：默认关闭。

作用：类似 KCP，用 QUIC 优化虚拟网内 TCP 流。高丢包场景可能更好，但不是必须。

### 禁用 QUIC 输入：`disable_quic_input`

建议：默认关闭。

作用：不接受别人用 QUIC 代理访问本节点。

### 禁用 P2P：`disable_p2p`

建议：不要开启。

作用：减少普通自动 P2P。你的目标是 P2P 低延迟，开启它会违背目标。

### 仅 P2P：`p2p_only`

建议：调试阶段关闭，稳定后可考虑开启。

作用：未建立 P2P 时不走 relay，直接丢包。符合“不依赖公网中转”的目标，但会降低可用性。建议先确认多地节点直连稳定后再开。

### 延迟 P2P：`lazy_p2p`

建议：节点很多时可开启，当前先关闭。

作用：有真实流量时才触发 P2P，减少空闲连接和打洞开销。节点少时没必要。

### 仅使用物理网卡：`bind_device`

建议：默认关闭。

作用：把连接绑定到物理设备，避免被其它虚拟路由/代理影响。你的本机有 Mihomo/198.18.x 代理环境时，这个选项可能有用，但需要知道具体网卡名，否则可能配错导致不可达。

### 无 TUN 模式：`no_tun`

建议：普通正式客户端关闭。

作用：不创建 TUN 虚拟网卡。无需 root，但不能像正常 VPN 一样直接 ping/访问路由。我们之前用它验证过控制面和 P2P，但正式客户端建议用 TUN。

### 启用出口节点：`enable_exit_node`

建议：普通客户端关闭。

作用：允许其它节点把全局流量从本机出口转发。会消耗本机带宽，也改变网络安全边界。

### 转发 RPC 包：`relay_all_peer_rpc`

建议：普通客户端关闭；共享/接入节点可开启。

作用：即使不转发业务流量，也帮助其它 peer 交换 RPC、发现和打洞。家庭公网节点适合开启，普通客户端不需要。

### 需要 P2P：`need_p2p`

建议：普通客户端关闭；服务器/游戏房主/固定入口节点可开启。

作用：声明本节点应被优先建立 P2P。

### 多线程：`multi_thread`

建议：开启。

作用：使用多线程运行时。桌面/服务器设备通常可以开启；低性能路由器可按负载调整。

### 系统转发：`proxy_forward_by_system`

建议：普通客户端关闭；站点网关节点按需开启。

作用：子网代理流量交给系统内核转发，禁用 EasyTier 内置 NAT。要求正确配置系统转发、防火墙和回程路由。普通客户端不需要。

### 禁用加密：`disable_encryption`

建议：不要开启。

作用：关闭 peer 通信加密。除非完全可信内网测试，否则不建议。

### 禁用 TCP 打洞：`disable_tcp_hole_punching`

建议：关闭。

作用：关闭 TCP 打洞。通常保留更多打洞方式更好。

### 禁用 UDP 打洞：`disable_udp_hole_punching`

建议：关闭。

作用：关闭 UDP 打洞。你的目标主要依赖 UDP P2P，不应开启。

### UDP 广播中继：`enable_udp_broadcast_relay`

建议：默认关闭。

作用：中继 UDP 广播，适合部分局域网发现协议/游戏发现。会增加广播流量，按需开启。

### 禁用 UPnP：`disable_upnp`

建议：默认关闭。

作用：禁止自动 UPnP/NAT-PMP 端口映射。普通 NAT 网络中，保留 UPnP 有助于直连；如果不信任局域网 UPnP 或路由器表现异常，可关闭。

### 禁用对称 NAT 打洞：`disable_sym_hole_punching`

建议：默认关闭。

作用：关闭针对对称 NAT 的端口预测/打洞。某些运营商可能阻断这种行为，但关闭会降低强 NAT 下的 P2P 成功率。

### 魔法 DNS：`accept_dns`

建议：默认关闭。

作用：让 EasyTier 修改系统 DNS，实现类似 `hostname.et.net` 的解析。可能影响本机 DNS/代理环境，确认需求后再开。

### 私有模式：`private_mode`

建议：开启。

作用：只允许相同网络名和密钥的节点通过本节点握手或中转，避免陌生网络借用你的节点。

## 推荐导入文件

通用客户端配置文件：

```text
/path/to/easytier-home-mesh-generic.toml
```

## 固定 IP 配置

如果设备需要固定虚拟 IP，不要使用通用配置直接复制给多台设备。应每台设备单独设置：

```toml
ipv4 = "10.144.1.X"
dhcp = false
```

建议规划：

```text
10.144.1.1   home-kwrt
10.144.1.10  local-gui
10.144.1.20  laptop
10.144.1.30  phone
10.144.1.100+ 临时设备 DHCP
```
